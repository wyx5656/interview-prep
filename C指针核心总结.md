# C指针核心知识点总结（面试版）

## 1. 指针基本概念

- 指针是**存放变量地址的变量**
- 32位系统占 4 字节，64位系统占 8 字节
- 指针运算：`+`/`-` 按**类型大小**跳跃，`int* +1` 跳 4 字节，`char* +1` 跳 1 字节

```c
int arr[5] = {10, 20, 30};
int *p = arr;
// p+1 指向 arr[1]，地址增加 sizeof(int)
```

## 2. 指针和数组

- 数组名是**数组首元素地址**，是一个**指针常量**，不能修改指向
- 四种访问方式等价：

```c
arr[0] = 10;
p[0] = 10;
*(p+0) = 10;
*(arr+0) = 10;
```

- `sizeof(arr)` = 整个数组的字节大小；`sizeof(p)` = 指针本身大小（8字节）

## 3. 常量指针 vs 指针常量

**口诀：const在*左是常量指针，const在*右是指针常量**

| 写法 | 名称 | *p能否修改 | p能否修改 |
|------|------|------------|-----------|
| `const int *p` | 常量指针（指向常量） | ❌ | ✅ |
| `int *const p` | 指针常量（指针本身常量） | ✅ | ❌ |

```c
int a = 10, b = 20;
const int *p = &a;  // 常量指针
// *p = 20;  // ❌ 不能改值
p = &b;            // ✅ 可以改指向

int *const p2 = &a; // 指针常量
*p2 = 20;          // ✅ 可以改值
// p2 = &b;       // ❌ 不能改指向
```

## 4. 野指针

**定义**：指针指向的内存已经释放/失效，但指针仍然保存那个地址

**常见原因**：
1. 返回局部变量地址（栈上变量出栈后失效）
2. `free` 释放内存后，指针没有置 `NULL`
3. 使用未初始化的野指针

```c
int* bad_function() {
    int x = 100;  // x在栈上
    return &x;    // ❌ 返回局部变量地址，x出栈后就是野指针
}
```

**避免方法**：
- `malloc/free` 后及时将指针置 `NULL`
- 不返回局部变量地址
- 定义指针时初始化为 `NULL`

## 5. 指针和字符串

```c
char* str = "hello";        // 指向文字常量区（只读），不能修改
// str[0] = 'a';  // ❌ 段错误

char str1[] = "hello";      // 在栈上，是常量区的拷贝，可以修改
str1[0] = 'a';  // ✅ 正常
```

## 6. 函数指针

**语法**：`返回类型(*指针名)(参数列表)`

**用途**：
- 回调函数
- 事件处理/状态机中存放函数指针
- 把函数当数据传递

```c
// typedef 简化类型定义
typedef void (*event_handler)();

// 结构体中存函数指针做事件映射
typedef struct{
    char *event_name;
    event_handler handler;
}EventMap;

// 回调函数示例
void callback(void(*func)()) {
    func();  // 动态调用
}

// 使用
EventMap events[] = {{"click", on_click}, {"hover", on_hover}};
events[0].handler(); // 调用对应函数
```

## 7. 常见面试题

### Q: sizeof 一个指针是多少？
A: 64位系统 8 字节，32位系统 4 字节，和指向什么类型无关。

### Q: `char str[] = "hello"` 和 `char *str = "hello"` 的区别？
A:
- `char str[]`: 栈上分配，内容可修改，是文字常量区的拷贝
- `char *str`: 指针指向文字常量区（只读），内容不可修改

### Q: 数组名能不能当左值？
A: 不能，数组名是常量，`arr = p` 编译错误。

### Q: 什么是回调函数？
A: 通过函数指针调用的函数，把函数指针作为参数传给另一个函数，当这个指针被用来调用其所指向的函数时，就是回调。
