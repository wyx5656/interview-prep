# CloudiskServer 项目重构总结

## 项目背景

原项目已经完成了基础功能，需要进行以下优化和重构：

1. **epoll 边缘触发优化** - 从水平触发改为边缘触发 + 非阻塞 Socket，提升高并发性能
2. **数据库连接池动态扩容** - 从固定大小改为支持动态扩容到上限，应对突发流量
3. **断点续传功能增强** - 修复下载偏移量计算 bug，给上传添加断点续传支持
4. **架构重构** - 按用户建议，删除双端口，改为单端口 + **主线程处理短任务，线程池处理长任务** 的简洁架构
5. **代码审查** - 全面检查并修复所有高危 bug
6. **性能测试** - 验证重构后的性能提升

## 架构设计

### 最终架构图

```
┌─────────────────────────────────────────────────────────────┐
│                    主线程 (epoll 循环)                          │
│  ┌─────────────┐                                            │
│  │  epoll_wait│  →  读取解析命令  →  判断长短任务            │
│  └─────────────┘                                            │
│        │                                                      │
│        ├─▶ 短任务：pwd/ls/cd/mkdir/rmdir/login/register       │
│        │        ▶  直接在主线程执行  →  快速响应                │
│        │                                                      │
│        └─▶ 长任务：puts/gets (上传下载)                        │
│                 ▶  入队到线程池  →  后台异步执行                │
└─────────────────────────────────────────────────────────────┘
           │                                                   
           ▼
┌─────────────────────────────────────────────────────────────┐
│               长任务线程池 (N 个工作线程)                      │
│         每个线程从队列取出任务，执行上传/下载                   │
│         完成后释放资源，继续等待下一个任务                      │
└─────────────────────────────────────────────────────────────┘
           │
           ▼
┌─────────────────────────────────────────────────────────────┐
│              数据库连接池 (动态扩容)                           │
│  初始连接 = thread_num, 最大连接 = max_thread_num             │
│  连接耗尽时自动扩容1个连接，直到最大值                         │
└─────────────────────────────────────────────────────────────┘
```

### 架构优势

| 优势 | 说明 |
|------|------|
| **低延迟** | 短任务直接在主线程执行，不需要线程池排队，响应极快 |
| **不阻塞** | 长任务交给线程池后台执行，主线程继续处理新请求，短任务不会被长任务阻塞 |
| **线程复用** | 长任务复用线程池线程，避免反复创建销毁线程的开销 |
| **架构简洁** | 单端口处理所有连接，删除了约 100 行无用代码 |
| **并发安全** | 全局共享数据都有互斥锁保护 |

### 代码审查修复高危问题

| 问题 | 修复方案 | 状态 |
|------|----------|------|
| Double free：`hashtable erase` 和 `deleteNode2` 都释放 `user_info_t` | 删除 `hashtable.c` 中的 `free`，只让 `deleteNode2` 释放 | ✅ 已修复 |
| 全局 `userTable` 无锁保护，并发访问存在竞态条件 | 添加 `pthread_mutex_t userTable_mutex` 保护所有读写操作 | ✅ 已修复 |
| 超时队列 `map` 数组只存储单个索引，哈希碰撞覆盖节点 | 重构 `map` 为链表存储，使用拉链法解决碰撞 | ✅ 已重构 |
| 用户正常断开连接不删除超时队列节点，导致内存泄漏 + 野指针 | 连接断开时调用 `removeUser` 从超时队列删除 | ✅ 已修复 |
| 哈希表 `MAX_SIZE = 100`，满了之后线性探测会死循环 | 增大到 1024，插入时添加探测次数限制 | ✅ 已修复 |
| daemonize `fork()` 调用遗漏，`pid` 一直为 0 | 添加 `pid = fork()` 调用 | ✅ 已修复 |

## 启动方式

```bash
# 进入 server 目录
cd /path/to/CloudiskServer4/server

# (可选) 设置文件存储路径，默认 ./files
export CLOUDISK_FILES="/path/to/your/files"

# 启动服务器（已经deamonize，自动后台运行）
./CloudiskServer ../conf/server.conf
```

## 配置说明 (`conf/server.conf`)

```
ip=127.0.0.1        # 监听地址
port=8080           # 监听端口（唯一端口）
thread_num=3        # 长任务线程池大小 = 初始数据库连接数
max_thread_num=20  # 数据库连接池最大容量（动态扩容上限）
```
