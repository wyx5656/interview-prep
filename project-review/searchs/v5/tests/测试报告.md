## 性能预期

基于项目架构和已有的优化，预期性能：

| 查询类型 | 缓存级别 | 预期延迟 |
|----------|----------|----------|
| 搜索推荐 | 一级缓存命中 | < 1 ms |
| 搜索推荐 | 二级缓存命中 | < 5 ms |
| 搜索推荐 | 缓存未命中 | < 20 ms |

| 全文搜索 | 二级缓存命中 | < 5 ms |
| 全文搜索 | 缓存未命中 | < 100 ms (取决于结果数量) |

**吞吐量预期：**
- 单核 QPS: 50-200
- 多核（8线程）QPS: 200-800
- 受限于 Redis 网络IO，实际取决于查询复杂度

### 使用方法

1. **启动服务器**: 
```bash
cd /home/wyx/interview-prep/project-review/searchs/v5
./bin/online
```

2. **测试搜索**: 
```bash
curl -X POST http://localhost:8888/search -H "Content-Type: application/json" -d '{"data":"中国"}'
```

3. **测试搜索推荐**: 
```bash
curl -X POST http://localhost:8888/suggest -H "Content-Type: application/json" -d '{"data":"se"}'
```

4. **运行单元测试**: 
```bash
./bin/test_edit_distance
./bin/test_double_lru
```

5. **运行基准测试**: 
```bash
python3 tests/benchmark.py
```

6. **运行并发压力测试**: 
```bash
python3 tests/test_concurrent.py --concurrency 16 --requests 1000
```
