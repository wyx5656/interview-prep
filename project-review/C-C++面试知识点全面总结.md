## 一、C语言核心知识点

### 1.1 static 关键字三种用法

```c
// 1. 修饰全局变量：作用域仅限于本文件，其他文件不能访问
static int global_counter = 0;

// 2. 修饰局部变量：存储在静态存储区，函数调用结束不销毁，值保留
static int get_next() {
    static int x = 0;
    return x++;
}  // 每次调用x递增，保留值

// 3. 修饰函数：函数只能在本文件调用，其他文件不能访问
static void helper_func() {
    // ...
}
```

### 1.2 #define 宏和函数区别

| 宏 | 函数 |
|----|------|
| 编译前预处理展开，不做类型检查 | 编译时类型检查 |
| 代码展开，没有函数调用开销 | 有调用压栈弹栈开销 |
| 可能有副作用（参数多次计算） | 参数只计算一次 |
| 比如 `#define MAX(a,b) ((a)>(b)?(a):(b))` | 内联函数可以兼顾性能和安全 |

**为什么宏定义常用 `do { ... } while(0)`？**
- 保证宏在任何上下文语法都正确，特别是在if语句中
- 比如宏里有多条语句，不用do-while会导致if只执行第一条，第二条脱离if控制

### 1.3 指针和数组区别

```c
int arr[5] = {1,2,3,4,5};  // arr是数组，分配5个int空间，arr是数组名代表首地址
int *p = arr;               // p是指针，存储arr首地址，p本身占8字节（64位）

sizeof(arr) = 5 * 4 = 20;   // 数组大小
sizeof(p) = 8;              // 指针大小
```

**总结：**
- 数组：内存分配了连续空间，数组名是首地址常量，不能修改
- 指针：存储别人的地址，指针变量本身占固定8字节（64位），可以指向不同地址

### 1.4 野指针是什么？怎么避免？

**野指针：** 指针指向一个已经释放的内存，或者指向未初始化的地址，访问会导致未定义行为（可能崩溃，可能修改不该改的内存）。

**避免方法：**
- 指针初始化 `= NULL`
- free之后指针置 `NULL`
- 不要返回栈变量的指针（栈变量函数返回就释放了）

### 1.5 内存分配 malloc/free 和 new/delete 区别

| malloc/free | new/delete |
|------------|------------|
| C语言标准库函数 | C++运算符 |
| 需要手动算 `malloc(n * sizeof(T))` | `new T` 自动算大小 |
| 返回 `void*` 需要强转 | 返回对应类型指针 |
| 不会调用构造/析构函数 | new调用构造，delete调用析构 |

**malloc分配内存失败返回什么？** → `NULL`

### 1.6 什么是缓冲区溢出？

写入数组超过了分配的大小，覆盖了相邻的内存，可能导致崩溃或者安全漏洞。

**怎么避免：** 每次写入检查边界，用 `strncpy` 代替 `strcpy`，`fgets` 代替 `gets`。

### 1.7 结构体对齐

**为什么对齐：** CPU存取内存按字（4/8字节）对齐访问更快，不对齐需要多次存取。

**规则：**
1. 第一个成员起始偏移 0
2. 每个成员对齐到自身大小的整数倍地址
3. 整个结构体大小对齐到最大成员大小的整数倍

**举例：**
```c
struct A {
    char a;   // 偏移 0，占1字节 → 填充3字节
    int b;    // 偏移 4，占4字节 → 总共 8 字节
};  // sizeof(A) = 8，不是5
```

**可以用 `#pragma pack(1)` 取消对齐，节省空间，但访问变慢。**

### 1.8 回调函数是什么？

回调函数就是函数指针，把函数指针传给别人，别人在合适时机调用你的函数。

**应用：** 事件处理，你注册一个回调函数，事件发生了系统调用你。

### 1.9 qsort 怎么用？

标准库快速排序，需要你提供一个比较函数：

```c
int compare(const void *a, const void *b) {
    int x = *(int*)a;
    int y = *(int*)b;
    return x - y;  // 升序
}
qsort(arr, n, sizeof(int), compare);
```

## 二、C++ 面向对象

### 2.1 面向对象三大特性

**封装：** 把数据和操作打包在一起，隐藏内部实现，只暴露接口 → 提高可维护性。

**继承：** 子类继承父类，可以复用父类代码，扩展功能。

**多态：** 子类重写父类方法，通过父类指针调用，实际调用子类版本 → 接口复用，不同子类不同行为。

### 2.2 多态怎么实现？（虚函数表）

**核心：** 虚函数表 + 虚指针

- 有虚函数的类，编译器会生成一张虚函数表（vtable），存虚函数地址
- 对象中会有一个虚指针（vptr）指向虚函数表
- 调用虚函数时，通过虚指针找到虚函数表，再找到函数地址调用 → 运行时确定调用哪个函数

**什么是覆盖（override）？** 子类重新实现父类的虚函数，虚函数表中子类地址覆盖父类地址。

### 2.3 构造函数能不能是虚函数？

**不能！** 原因：
- 构造对象的时候，虚指针还没完全初始化好，不能查表
- 构造阶段类型还没确定，虚函数也没意义

**析构函数能不能是虚函数？**
- **应该！** 如果你要通过父类指针删除子类对象，父类析构必须虚，否则只会调用父类析构，子类部分不析构 → 内存泄漏

### 2.4 什么是切片（object slicing）？

```cpp
class Animal { ... };
class Dog : public Animal { ... };

Dog d;
Animal a = d;  // 切片！只拷贝父类部分，子类部分切掉了
```

**避免：** 要多态用指针或引用。

### 2.5 重载和重写区别

| 重载（overload） | 重写（override） |
|-----------------|-----------------|
| 同一个类，同一个方法名，不同参数列表 | 子类继承父类，子类重新实现父类虚函数 |
| 编译时根据参数类型确定调用哪个 | 运行时根据实际类型确定调用哪个 |
| 不覆盖，共存 | 覆盖父类版本 |

### 2.6 深拷贝和浅拷贝

**浅拷贝：** 只拷贝指针，不拷贝指针指向的内存 → 两个对象指向同一块内存，double free → 错了！

**深拷贝：** 拷贝指针指向的内存，新对象有自己的副本 → 没问题。

```cpp
class String {
    char *data;
public:
    // 深拷贝拷贝构造
    String(const String &other) {
        data = new char[strlen(other.data) + 1];
        strcpy(data, other.data);  // 真正分配内存拷贝
    }
};
```

**默认拷贝构造是浅拷贝！** 如果类里有动态分配内存，必须自己写拷贝构造和赋值运算符（三法则：析构/拷贝构造/赋值）。

### 2.7 解释一下虚继承，解决什么问题？

**解决菱形继承的二义性问题：**

```
    A
   / \
  B   C
   \ /
    D
```

D继承B和C，B和C都继承A，D里有两份A，访问A成员会有二义性。

**虚继承：** B虚继承A，C虚继承A，D里只保留一份A → 解决二义性。

### 2.8 什么是RAII？

**Resource Acquisition Is Initialization** 资源获取即初始化。

**思想：** 在构造函数获取资源，析构函数释放资源。

**好处：** 异常安全，离开作用域自动释放，不用手动释放，不会漏。

**例子：**
- `std::lock_guard<mutex>` 构造加锁，析构解锁
- `std::unique_ptr` 构造持有指针，析构delete
- fopen打开文件，RAII类析构fclose

## 三、C++ 新特性（C++11及以后）

### 3.1 `auto` 关键字

**自动推导类型，不用写长长的类型名：**
```cpp
auto it = map.begin();  // 不用写 std::map<...>::iterator
auto lambda = [](){...};
```

**注意：** auto不能推导数组类型，函数参数不能用auto（C++14以后可以）。

### 3.2 `nullptr` 代替 `NULL`

`nullptr` 是真正的空指针，类型是 `nullptr_t`，不会和整数混淆，类型安全。

### 3.3 `=default` `=delete`

```cpp
// =default：要求编译器生成默认版本，你写了其他构造编译器就不生成默认了，要就加=default
A() = default;

// =delete：禁止拷贝
A(const A&) = delete;
A& operator=(const A&) = delete;
```

### 3.4 智能指针 `unique_ptr` `shared_ptr` `weak_ptr`

| 智能指针 | 所有权 | 引用计数？ | 用途 |
|----------|--------|-----------|------|
| `unique_ptr` | 独占所有权 | 没有 | 独占对象，离开作用域自动释放，不能拷贝只能移动 |
| `shared_ptr` | 共享所有权 | 有，原子计数 | 多个所有者共享对象，计数到0自动释放 |
| `weak_ptr` | 没有所有权 | 不增加计数 | 观察shared_ptr，不影响生命周期，解决循环引用 |

**循环引用问题：** A有shared_ptr<B>，B有shared_ptr<A>，计数永远到不了0，永远不释放 → 解决：把一个改成weak_ptr。

**我的项目用在哪里？** 搜索引擎双缓存LRU里，`unique_ptr` 持有线程缓存。

### 3.5 `lambda` 表达式

```cpp
// [捕获] (参数) -> 返回值 { 体 }
sort(v.begin(), v.end(), [](int a, int b) { return a < b; });
```

捕获：
- `[=]` 值捕获所有
- `[&]` 引用捕获所有
- `[this]` 捕获this指针

**用途：** 回调，短小函数，不用到处写命名函数。

### 3.6 `std::function` 和 `std::bind`

- `std::function` 可以存储任意可调用对象（函数指针、lambda、仿函数），方便实现回调
- `std::bind` 把函数和参数绑定，生成新的可调用对象

**项目中：** 搜索引擎线程池任务队列存 `std::function<void()>`。

### 3.7 `thread_local` 线程局部存储

```cpp
static thread_local RedisSingleton instance;
```

**每个线程有一个独立实例，其他线程访问不到 → 我的项目用：**
1. 每个线程一个Redis连接，解决hiredis非线程安全问题
2. 每个线程私有缓存，读完全无锁

### 3.8 右值引用和移动语义

**左值：** 可以取地址，有名字，比如变量。
**右值：** 不能取地址，临时对象。

**右值引用 `T&&` 绑定右值，** 可以把资源从一个对象"移动"到另一个，避免拷贝，提升性能。

**举例：** `vector` 插入临时对象，移动构造比拷贝构造快很多。

**`std::move` 干什么？** 把左值转换成右值引用，触发移动。

### 3.9 `std::atomic` 原子类型

不用加锁就能线程安全访问，适合简单计数器、标志位。

```cpp
std::atomic<bool> running{true};
std::atomic<int> counter{0};
```

### 3.10 `range-based for` 范围for循环

```cpp
for (auto &x : vec) {
    cout << x << endl;
}
```

比传统for简洁，不容易写错下标。

### 3.11 `override` `final` 关键字

- `override`：显式标记重写父类虚函数，写错签名编译报错，提前发现错误
- `final`：类不能被继承，虚函数不能被重写

### 3.12 可变参数模板

模板可以接受任意个数参数，比如 `std::tuple`，`printf` 现代化实现。面试能说清楚就行。

## 四、内存管理

### 4.1 C/C++ 内存分配区域

| 区域 | 放什么 |
|------|--------|
| **栈** stack | 局部变量、函数参数、返回地址，自动分配自动释放，大小固定 |
| **堆** heap | 动态分配内存，malloc/new出来的，用户管理释放，自由大小 |
| **静态存储区** | 全局变量、static变量，程序运行整个生命周期存在 |
| **常量区** | 字符串常量、const常量，只读 |
| **代码区** | 存放二进制指令，只读 |

### 4.2 内存泄漏是什么？

**malloc/new出来，忘记free/delete，程序运行一直不释放，内存越来越少，就是泄漏。**

**怎么检测：** valgrind工具检测，代码review看有没有new不delete。

### 4.3 野指针和空指针区别

- **空指针：** 指向NULL，地址0，访问肯定崩溃，好调试
- **野指针：** 指向一块已经释放的内存或者随机地址，可能跑过去，可能随机崩溃，难调试

### 4.4 free/delete 后指针怎么处理？

立即置为 `NULL`，避免野指针，后面访问判空就能保护。

### 4.5 delete 和 delete[] 区别

- `new` 对应 `delete`
- `new T[]` 数组对应 `delete[]` → 用错了结果未定义，数组不会正确调用所有元素析构

### 4.6 内存碎片怎么产生？怎么解决？

**产生：** 频繁分配释放不同大小内存，剩下很多小块空闲内存没办法分配大对象 → 碎片。

**解决：**
- 内存池：预先分配一块大内存，自己管理，用完不释放还给池，减少碎片
- 固定大小块分配：相同大小对象从对应块分配，不会产生碎片

## 五、指针和引用

### 5.1 指针和引用区别

| 指针 | 引用 |
|------|------|
| 是个变量，存储地址，可以为空，可以改变指向 | 是变量别名，不能为空，定义必须初始化，初始化后不能改变指向别的变量 |
| 多级指针 | 没有多级引用 |
| sizeof指针是8字节（64位） | sizeof引用是引用对象大小 |

### 5.2 为什么函数参数要用引用？什么情况用指针？

**引用优点：** 不用拷贝大对象，节省时间空间，语法比指针简洁。
**必须用指针情况：** 可以为空，需要改变指向。

### 5.3 引用能空吗？标准说不能，但是可以写出来吗？
```cpp
int *p = nullptr;
int &r = *p;  // 语法能编译，但访问未定义行为，崩溃
```
实际不要这么写。

## 六、多线程并发

### 6.1 什么是线程安全？

多个线程同时访问共享数据，结果还是正确的，就是线程安全。

### 6.2 怎么保证线程安全？

- 互斥锁 `pthread_mutex_t` / `std::mutex` → 同一时间只有一个线程进入临界区
- 原子变量 `std::atomic` → 简单操作不用锁
- 读写锁 `pthread_rwlock_t` → 读多写少，多个读者可以同时进，提高并发

### 6.3 什么是竞态条件？

多个线程执行顺序不同结果不同，因为调度顺序不确定导致错误。解决就是加锁。

### 6.4 什么是死锁？四个必要条件？

**死锁：** 多个线程互相拿着对方要的锁，都卡住不动，永远等下去。

**四个必要条件：**
1. **互斥：** 资源不能共享，只能一个线程用
2. **占有并等待：** 拿着一个锁等待另一个
3. **不可抢占：** 不能强行从别人手里抢锁
4. **循环等待：** 循环等待对方占有的锁

**破坏任意一个条件就不会死锁。**

**怎么避免死锁：**
- 按相同顺序加锁，比如都先锁A再锁B，不会循环等待
- 尽量减少锁的范围和个数
- 用try_lock，拿不到就释放已经拿的，重试

### 6.5 生产者消费者模型

**一个缓冲区，生产者放任务，消费者取任务。**
- 空缓冲区，消费者等
- 满缓冲区，生产者等
- 用 互斥锁 + 条件变量 实现

**代码框架：**
```cpp
mutex m;
condition_variable cv;
queue<T> q;

// 生产者
void producer() {
    while (true) {
        unique_lock<mutex> lock(m);
        while (q.full()) {
            cv.wait(lock);  // 缓冲区满，等待
        }
        q.push(item);
        cv.notify_one();  // 通知消费者有货了
    }
}

// 消费者
void consumer() {
    while (true) {
        unique_lock<mutex> lock(m);
        while (q.empty()) {
            cv.wait(lock);  // 缓冲区空，等待
        }
        auto item = q.front(); q.pop();
        cv.notify_one();  // 通知生产者有空位了
        // 处理item
    }
}
```

**为什么 while 循环等待不用 if？** → 虚假唤醒！多个消费者同时唤醒，一个拿走了，另一个醒来队列又是空，所以必须再判断。

我的两个项目都用了生产者消费者：
- Cloudisk：主线程生产任务，线程池消费者拿任务执行
- 搜索引擎：Reactor主线程生产任务，线程池执行

### 6.6 什么是惊群效应？

多个线程都阻塞在accept，一个新连接进来，内核唤醒所有线程，但只有一个accept成功，其他都失败，浪费资源，就是惊群。

**解决：** 只有一个线程accept，我的项目就是这样，只有主线程epoll accept。

### 6.7 thread_local 用途

每个线程一份实例，我的搜索引擎项目：
- 每个线程一个Redis连接，解决hiredis非线程安全，不用加锁
- 每个线程一级缓存，读完全无锁，性能好

## 七、设计模式

### 7.1 单例模式

**一个类只能创建一个实例，全局访问。**

**C++11 写法：**
```cpp
class Singleton {
public:
    static Singleton* getInstance() {
        static Singleton instance;  // C++11静态局部变量初始化线程安全
        return &instance;
    }
    // 禁止拷贝
    Singleton(const Singleton&) = delete;
    Singleton& operator=(const Singleton&) = delete;
private:
    Singleton() {}  // 私有构造，不能外面创建
};
```

**我的项目用在哪里：** 搜索引擎词典单例，全局只有一个词典。

### 7.2 工厂模式

**简单工厂：** 一个工厂类根据参数创建不同产品
**工厂方法：** 每个产品对应一个工厂，子类决定创建什么
**抽象工厂：** 创建一系列相关产品

### 7.3 观察者模式

**一个对象变化，通知所有观察者自动更新 → 事件驱动。**

### 7.4 策略模式

**不同算法封装成不同策略类，接口统一，运行时换算法。**

### 7.5 装饰器模式

**不改变原类，动态给对象增加功能，比继承灵活。**

## 八、常用数据结构

### 8.1 数组 vs 链表

| 数组 | 链表 |
|------|------|
| 连续内存，随机访问O(1) | 不连续，顺序访问，访问O(n) |
| 插入删除要移动元素，O(n) | 插入删除只要改指针，O(1)（知道位置） |
| 空间固定，满了要扩容 | 动态分配，不用担心扩容 |
| 缓存友好，连续内存命中率高 | 不连续，缓存命中率低 |

### 8.2 哈希表

**原理：** 哈希函数把key映射到数组索引，O(1)查找。

**哈希冲突解决：**
- 链地址法：每个桶挂链表，我的项目用这个
- 开放寻址法：线性探测往下找空位置

### 8.3 LRU 缓存淘汰

**需求：** 缓存满了，淘汰最久未使用。

**数据结构：** 双向链表 + 哈希表

- 双向链表：按访问顺序排，表头最近访问，表尾最久未用
- 哈希表：O(1)找节点

**操作：**
- 访问命中：把节点移到表头
- 插入：插到表头，如果容量满了删除表尾
- 我的搜索引擎项目，双缓存LRU就是这么实现的。

### 8.4 二叉搜索树 BST

性质：左子树都比根小，右子树都比根大 → 中序遍历是升序。

查找：平均O(logn)，最坏退化成链表O(n) → 所以有AVL、红黑树。

### 8.5 红黑树性质

- 每个节点红或黑
- 根黑，NIL叶子黑
- 红节点两个孩子都是黑
- 任意节点到每个叶子路径黑节点个数相同

**平衡不严格，插入删除旋转比AVL少，性能好，C++ std::map 就是红黑树。**

### 8.6 堆 大顶堆小顶堆

- 大顶堆：父节点大于等于孩子，顶部最大 → 求前K小用大顶堆
- 小顶堆：父节点小于等于孩子，顶部最小 → 求前K大用小顶堆
- 建堆O(n)，插入删除O(logn)

### 8.7 排序算法总结

| 排序 | 平均时间 | 最坏 | 空间 | 稳定？ |
|------|---------|------|------|--------|
| 冒泡 | O(n²) | O(n²) | O(1) | 稳定 |
| 选择 | O(n²) | O(n²) | O(1) | 不稳定 |
| 插入 | O(n²) | O(n²) | O(1) | 稳定 |
| 快排 | O(nlogn) | O(n²) | O(logn)栈 | 不稳定 |
| 归并 | O(nlogn) | O(nlogn) | O(n) | 稳定 |
| 堆排 | O(nlogn) | O(nlogn) | O(1) | 不稳定 |

**快排基准选择：** 随机选，三数取中，避免最坏O(n²)。

**快排优化：** 小数组插入排序，尾递归优化。

## 十、编译链接过程

### 10.1 四个步骤

```
hello.c → [预处理] → hello.i → [编译] → hello.s → [汇编] → hello.o → [链接] → hello可执行文件
```

**预处理：** 展开宏，展开头文件，去掉注释，处理#include #define #if这些。

**编译：** C代码翻译成汇编代码。

**汇编：** 汇编代码翻译成机器指令（二进制），生成目标文件.o。

**链接：** 把多个.o和库文件链接在一起，合并符号，重定位，生成可执行文件。

### 10.2 静态链接 vs 动态链接

**静态链接：** 把库代码复制到可执行文件 → 文件大，运行不用依赖库，启动快。
**动态链接：** 只链接信息，运行的时候加载库 → 文件小，多个程序共享库内存，更新库不用重新编译链接。

### 10.3 什么是符号表？

存函数和全局变量名字和地址，链接时候需要符号解析，找到引用对应地址。

### 10.4 Makefile 作用

**自动化编译，描述哪些文件要编译，依赖关系，怎么编译，make自动执行。**

## 十一、常见面试问题Q&A

### Q1: 宏和内联函数区别？

**A：**
- 宏是预处理展开，不做类型检查
- 内联函数是编译器 inline，有类型检查，更安全
- 内联会把代码展开到调用处，省去函数调用开销，和宏一样快，比宏安全

### Q2: 为什么拷贝构造函数参数必须是引用？不能传值？

**A：** 传值要拷贝实参给形参，调用拷贝构造，无穷递归了！所以必须引用。

### Q3: 什么是内存对齐，为什么要对齐？

**A：** 上面说了，CPU按字对齐访问更快，不对齐需要多次读写，所以编译器自动对齐。

### Q4: inline 关键字干什么？一定会内联吗？

**A：** inline给编译器建议展开函数，省去调用开销，但是编译器可以不听你的，大函数不会内联。inline不代表一定内联，只是建议。

**inline内联函数和普通函数区别：** 链接的时候，多个编译单元同一个inline函数不会冲突，链接器合并成一个。

### Q5: 什么是未定义行为？

**A：** C/C++标准没规定结果，比如访问越界数组，解引用空指针，销毁后引用，结果不确定，可能崩溃可能乱码可能正常跑，难调试。

### Q6: 什么时候用指针，什么时候用引用？

**A：**
- 必须不空，绑定后不换指向 → 用引用，语法简洁
- 可以空，或者需要换指向 → 用指针

### Q7: shared_ptr 线程安全吗？

**A：** 引用计数更新是原子的，线程安全。但是对象本身的访问不是线程安全，要自己加锁。

### Q8: 怎么解决shared_ptr循环引用？

**A：** 把其中一个改成weak_ptr，不增加引用计数，就能释放了。

### Q9: epoll LT 和 ET 区别？

**A：** 我的云盘项目讲过：
- LT水平触发：只要缓冲区有数据就一直通知
- ET边缘触发：只有状态从无变有才通知一次，减少系统调用，性能更好，必须非阻塞循环读到EAGAIN

### Q10: 什么是零拷贝？

**A：** 减少用户态内核态之间的数据拷贝，比如mmap，sendfile，比read/write少一次拷贝，性能更好。我的云盘上传下载用mmap零拷贝。
